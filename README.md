# HopTerm

**HopTerm** — GUI-менеджер SSH-подключений и терминал для Linux. Держит все ваши
хосты в одном месте, ходит на них через цепочки jump-серверов, даёт живой
терминал, передаёт файлы по SFTP и запускает сохранённые команды — в одном
приложении с современным интерфейсом.

## Возможности

- **Многохоповые маршруты.** `локальная машина → jump1 → jump2 → … → целевой хост`
  без ограничения на число хопов. Терминал и передачи файлов идут через всю
  цепочку прозрачно.
- **Живой терминал.** Полноценный интерактивный shell (`xterm.js` поверх
  настоящего PTY): цвета, TUI-приложения, ресайз под окно, копирование/вставка
  `Ctrl+C`/`Ctrl+V` в любой раскладке клавиатуры.
- **Несколько сессий сразу.** Каждый хост — своя вкладка терминала; переключение
  без переподключения, сессии открываются лениво.
- **Аутентификация на выбор:** SSH-ключ, пароль (можно сохранить в конфиг),
  SSH-agent. Отдельная кнопка **«Подключиться с sudo»** с выбором способа
  эскалации (`sudo -s`, `sudo su`, `su`, …) и паролем.
- **Менеджер хостов.** Список подключений с тегами и фильтрами по ним,
  быстрый доступ к недавним, редактор хоста с конструктором цепочки хопов.
- **Передача файлов по SFTP** через ту же цепочку: браузер удалённой ФС,
  загрузка и скачивание с прогресс-барами и отменой. Скачивание **по маске**
  (`*`, `?`) упаковывает всё совпавшее в архив, передаёт потоком и распаковывает
  локально — одна задача с общим прогрессом.
- **Конструктор команд.** Сохранённые быстрые команды четырёх типов —
  *Выполнить*, *Скачать*, *Загрузить*, *Пайплайн* — с живым предпросмотром
  (подсветка синтаксиса) и опцией подтверждения перед запуском.
- **Безопасность соединений.** Проверка host-key при подключении: сверка с
  `known_hosts`, TOFU при первом контакте, разрыв при несовпадении (защита от
  MITM). Секреты не хранятся в логах.
- **Тёмная и светлая темы.**

## Установка

### Из `.deb` (Debian/Ubuntu)

```bash
sudo apt install ./dist/hopterm_0.1.1-1_amd64.deb
```

`apt` подтянет зависимости (`libwebkit2gtk-4.1-0`, GTK 3). После установки
приложение доступно в меню как **HopTerm** и командой `hopterm`.

Собрать пакет самостоятельно:

```bash
packaging/make-deb.sh          # → dist/hopterm_<версия>-<rev>_<арх>.deb
```

Подробнее об опциях сборки — [`packaging/README.md`](packaging/README.md).

### Из исходников

Нужны Rust (1.80+) и системные библиотеки webview:

```bash
sudo apt install libwebkit2gtk-4.1-dev libgtk-3-dev   # один раз
cargo build --release -p hopterm-ui
./target/release/hopterm
```

## Быстрый старт

1. **Добавьте хост** — кнопка «＋ Добавить хост». Укажите имя, теги и цепочку:
   целевой хост плюс, при необходимости, jump-серверы (каждый со своим
   пользователем, портом и способом аутентификации).
2. **Подключитесь** кликом по хосту — откроется вкладка терминала. Кнопка
   «с sudo» открывает отдельную сессию с эскалацией привилегий.
3. **Передавайте файлы** на вкладке «Передачи»: перейдите по удалённой ФС и
   скачайте файл (или маску вроде `/var/log/app/*` — приедет одним архивом),
   либо загрузите локальный файл в текущую папку.
4. **Сохраняйте команды** на вкладке «Команды» через конструктор и запускайте их
   на нужном хосте одной кнопкой.

## Конфигурация

Всё хранится в `~/.hopterm/`:

| Файл | Содержимое |
|------|-----------|
| `config.toml` | Хосты, цепочки хопов, настройки |
| `commands.json` | Сохранённые быстрые команды |
| `known_hosts` | Закреплённые host-key (TOFU) |

Скачанные файлы по умолчанию попадают в `~/Downloads` (или в папку, заданную
в команде скачивания).

## Под капотом

Бэкенд целиком на Rust: SSH-транспорт — [`russh`](https://crates.io/crates/russh)
(многохоповые цепочки через `direct-tcpip`), SFTP — `russh-sftp`, VT/ANSI-движок —
`alacritty_terminal`. Интерфейс — системный webview ([`wry`](https://crates.io/crates/wry))
с терминалом на `xterm.js`; GUI общается с бэкендом по JSON-IPC и ничего не знает
о деталях SSH. Проект — cargo-workspace из отдельных крейтов по слоям.
